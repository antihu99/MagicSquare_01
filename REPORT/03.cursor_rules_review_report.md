# Cursor Rules 검토 보고서

## 개요

MagicSquare 프로젝트의 Java 기준 Cursor Rules `.mdc` 파일 7개를 검토했다.

검토 대상은 `.cursor/rules/` 폴더의 rules 파일이며, 요청된 기준에 따라 문법,
필수 구성, 규칙 충돌, AI 행동 규칙의 실행 가능성을 확인했다.

## 검토 대상

- `.cursor/rules/project.mdc`
- `.cursor/rules/java-code-style.mdc`
- `.cursor/rules/ecb-architecture.mdc`
- `.cursor/rules/tdd-rules.mdc`
- `.cursor/rules/testing.mdc`
- `.cursor/rules/forbidden-patterns.mdc`
- `.cursor/rules/ai-behavior.mdc`

## 검토 항목

### 1. YAML 문법 오류

문제 없음.

각 `.mdc` 파일의 frontmatter는 `---`로 정상 구분되어 있으며, `description`,
`globs`, `alwaysApply` 필드 구조에 문법상 문제는 발견되지 않았다.

### 2. 누락된 필수 섹션

문제 없음.

Java 기준 Cursor Rules는 다음 주제로 분리되어 있으며, 기존 `.cursorrules`의 주요
섹션을 모두 대체한다.

- 프로젝트 개요
- Java 코드 스타일
- ECB 아키텍처
- TDD 규칙
- 테스트 규칙
- 금지 패턴
- AI 행동 규칙

### 3. TDD 규칙과 금지 규칙 간 충돌

문제 없음.

TDD 규칙은 Red, Green, Refactor 단계의 작업 흐름을 정의하고, 금지 규칙은 Java
구현에서 피해야 할 패턴을 정의한다. 두 규칙 사이에 서로 반대되는 지시는 없다.

### 4. AI 행동 규칙 실행 가능성

문제 없음.

`ai-behavior.mdc`의 규칙은 Cursor AI가 실제 작업 중 확인하거나 수행할 수 있는
행동으로 구성되어 있다.

- 관련 테스트 파일 확인
- 테스트 우선 작성 또는 제안
- ECB 레이어 식별
- Java 메서드 타입 명시
- 의존성 방향 준수
- TDD 규칙 충돌 시 경고
- 관련 JUnit 테스트 실행 또는 제안

## 최종 결과

검토 결과, 현재 Java 기준 Cursor Rules `.mdc` 파일에서 보고할 문제점은 없다.
