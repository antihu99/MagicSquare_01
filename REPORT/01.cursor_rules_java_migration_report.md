# MagicSquare Cursor Rules Java Migration Report

## Overview

This report summarizes the conversion of the MagicSquare project rules from a
Python-oriented `.cursorrules` file into Java-oriented Cursor project rules.

The new rules are split into focused `.mdc` files under `.cursor/rules/` so that
Cursor AI can apply them by topic and file scope.

## Created Rule Files

- `.cursor/rules/project.mdc`
  - Defines the MagicSquare project goal, domain, and Java language basis.

- `.cursor/rules/java-code-style.mdc`
  - Defines Java 17+, Google Java Style, explicit method types, Javadoc, and
    immutable entity preferences.

- `.cursor/rules/ecb-architecture.mdc`
  - Defines the ECB layers: `boundary`, `control`, and `entity`.
  - Specifies the allowed dependency direction: `boundary -> control -> entity`.

- `.cursor/rules/tdd-rules.mdc`
  - Defines Red, Green, and Refactor phase rules for test-driven development.

- `.cursor/rules/testing.mdc`
  - Defines JUnit 5, AAA test structure, naming conventions, fixture rules, and
    minimum coverage expectations.

- `.cursor/rules/forbidden-patterns.mdc`
  - Defines forbidden Java patterns such as uncontrolled `System.out.println()`,
    hard-coded constants, broad exception handling, empty catch blocks, and
    globally mutable domain state.

- `.cursor/rules/ai-behavior.mdc`
  - Defines required Cursor AI behavior before, during, and after code changes.

## Main Changes From Python Rules

- Python version rules were replaced with Java 17+ rules.
- PEP8 and Black conventions were replaced with Google Java Style.
- Python type hints were replaced with explicit Java parameter, return, and field
  types.
- Google-style Python docstrings were replaced with Java Javadoc requirements.
- `pytest` testing rules were replaced with JUnit 5 testing rules.
- Python-specific forbidden patterns were replaced with Java-specific forbidden
  patterns.

## ECB Architecture Summary

The Java project should use these package-level responsibilities:

- `boundary`: Handles CLI, UI, API, and other external input/output.
- `control`: Coordinates business use cases and invokes entity rules.
- `entity`: Owns domain data and domain behavior.

Allowed dependency direction:

```text
boundary -> control -> entity
```

Forbidden dependency direction:

```text
entity -> control
entity -> boundary
control -> boundary
```

## Current Project Notes

- The original `.cursorrules` file still exists and contains Python-oriented
  rules.
- Python sample files also still exist under `entity/` and `tests/`.
- The Java rules are now available under `.cursor/rules/`.

## Recommendations

- Decide whether to keep or remove the old Python `.cursorrules` file.
- Replace the existing Python sample code with Java source and test structure if
  the project is now Java-based.
- Use a Java build tool such as Gradle or Maven before implementing the main
  MagicSquare features.
- Add JUnit 5 configuration before writing new Java tests.
