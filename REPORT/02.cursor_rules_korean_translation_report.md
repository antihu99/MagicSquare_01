# Cursor Rules 한글화 보고서

## 개요

MagicSquare 프로젝트의 Java 기준 Cursor Rules 파일 7개를 영어에서 한국어로
변환했다.

규칙 파일의 Cursor 메타데이터 구조는 유지하되, 사람이 읽는 설명 문구와 본문
규칙을 한국어로 작성했다.

## 대상 파일

- `.cursor/rules/project.mdc`
- `.cursor/rules/java-code-style.mdc`
- `.cursor/rules/ecb-architecture.mdc`
- `.cursor/rules/tdd-rules.mdc`
- `.cursor/rules/testing.mdc`
- `.cursor/rules/forbidden-patterns.mdc`
- `.cursor/rules/ai-behavior.mdc`

## 변경 내용

- 각 rules 파일의 `description` 값을 한국어로 변경했다.
- 제목과 본문 규칙을 한국어 문장으로 변경했다.
- Java 버전, JUnit 5, Google Java Style, ECB 의존성 방향 같은 핵심 규칙은
  유지했다.
- `globs`, `alwaysApply`, 파일 경로, 패키지명, 코드 식별자는 기존 의미가
  변하지 않도록 유지했다.

## 유지한 규칙 구조

- 프로젝트 개요 규칙은 항상 적용되도록 유지했다.
- Java 코드 스타일 규칙은 `**/*.java` 범위에 적용되도록 유지했다.
- ECB 아키텍처 규칙은 `boundary -> control -> entity` 의존성 방향을 유지했다.
- TDD 규칙은 Red, Green, Refactor 단계로 유지했다.
- 테스트 규칙은 JUnit 5와 AAA 패턴을 기준으로 유지했다.
- 금지 패턴 규칙은 Java 코드에서 피해야 할 구현 방식을 명시하도록 유지했다.
- AI 행동 규칙은 코드 작성 전, 중, 후에 따라야 할 작업 흐름을 유지했다.

## 검증 결과

- `.cursor/rules/` 폴더에 대한 린트 진단 결과 오류는 없었다.
- 한글화 이후에도 Cursor Rules 파일 형식인 `.mdc` 구조는 유지되었다.

## 후속 권장 사항

- 기존 Python 기반 `.cursorrules` 파일을 계속 유지할지 결정해야 한다.
- 프로젝트가 Java 기준으로 전환되었다면 Python 예제 코드와 테스트를 Java 구조로
  정리하는 것이 좋다.
- Java 빌드 도구로 Gradle 또는 Maven을 선택한 뒤 JUnit 5 테스트 환경을 구성하는
  것이 좋다.
